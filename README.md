GDB Dojo will be a hands-on GDB learning path for real software debugging.

The goal is to help C and C++ programmers build practical debugger skill, starting with first-time GDB use and growing into crash analysis, memory inspection, core dumps, optimized builds, binary inspection, and threaded programs.

This repository is early. Exercises and tooling have not been added yet.

## Scope

Planned topics:

- breakpoints and stepping
- variables, stack frames, and backtraces
- crashes and core dumps
- memory lifetime and buffer-boundary bugs
- optimized-build debugging
- binary inspection
- threads

All future practice content will be designed for local debugging scenarios.
